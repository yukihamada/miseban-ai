# カスタマーインタビューガイド - MisebanAI
## 5件のお客様インタビュー前実施ガイド

**作成日**: 2026年2月
**対象**: 飲食店2件、小売2件、サービス業1件
**目的**: 広告費を使う前にプロダクト仮説を検証

---

## 1. インタビュー目的

### プライマリー仮説
「店舗経営者は来客数データを経営判断に使うことで、人件費・仕入れ・営業施策の効率化ができる」

### バリデーション目標
5件中**3件以上**が以下のいずれかに言及
- 来客数・来客パターンの把握が経営上の課題
- 現在のやり方（目視・想像・簡易計数）では不十分
- カメラの既設設備を活用できる価値を認識
- 月額料金に対して購買意欲を示唆

### 失敗シナリオ（GO/NO-GO判断）
- **GO**: 3件以上が来客データへのニーズを言及 → 広告テスト開始
- **NO-GO**: 2件以下のみ → プロダクト改善・別セグメント検討

## 2. インタビュー進行ガイド（30分想定）

### 2-1. 開始前の準備（3分）

```
挨拶＋背景説明：
「本日はお忙しいところありがとうございます。
弊社は『カメラ映像をAIで分析して店舗データを提供するサービス』を開発中です。
実装前に、実際の店舗経営の課題をお聞かせいただき、本当に役に立つものを作りたくて。
本日の内容は社内の改善参考にさせていただきます。」

（アイスブレイク：天気、営業終わり具合など軽く）
```

### 2-2. 店舗概要（3分） - 関係構築フェーズ

**目的**: リラックスさせる + 基本情報を把握

```
1. 業種：何を扱っているお店ですか？
   → 飲食店なら：ラーメン？カフェ？居酒屋？など詳細化

2. 規模：
   - スタッフ数は？
   - 営業日・営業時間は？
   - 1日の来客数はざっくり何人くらい？

3. 最近の状況：
   - 最近、営業に変化はありましたか？（GW前後、季節変動など）
   - 去年との比較でどうですか？
```

**メモ例**: 「渋谷のラーメン屋、スタッフ5名、1日80〜150人、最近テレワーク利用客が減った」

---

### 2-3. ペイン質問（5分） - 最重要セクション ⭐⭐⭐

**注意**: 誘導的にならない。来客数の話を先に出さない

```
【開放型質問】（まずはこれ）
「今、この店舗の経営で一番困っていることってありますか？」

（相手の回答を聞く。来客数が出ない場合も進める）

【掘り下げ】
もし来客数と関係のない悩みが出た場合：
「なるほど。それって、例えば『今月は誰が休みが多いから営業が落ちる』みたいな
 来客数の把握と関係しているんですか？」
→ 間接的に来客データとの関連性を探る

もし来客数が出た場合：
「来客数がわかると、どうなると良いですか？」
→ 何に使いたいのかを掘る（シフト計画、仕入れ、営業施策など）
```

**クリティカル**: 相手の言葉から「本当のペイン」を引き出す
- 「人件費が大きい」← 来客数で効率化できる可能性
- 「仕入れが難しい」← 来客予測で在庫最適化できる可能性
- 「営業成果がわかりにくい」← キャンペーン効果測定ができる可能性

---

### 2-4. 現在の行動（4分） - 現状把握

```
「今、来客数をどうやって把握していますか？」

パターン別対応：

1. 「特に把握していない」
   → 「シフト計画や仕入れはどうやって決めてるんですか？」
   → 「経験則？」「スタッフの勘？」などを引き出す

2. 「目視で数えてる」
   → 「誰が数える？いつ？」「手書きしてる？」
   → 「それって時間かかりますか？」

3. 「売上で推測」
   → 「客単価はどのくらい変わる？」「その方法で十分？」

4. 「簡易カウンター使ってる」
   → 「どのくらいの精度だと思う？」「時間帯別に分けてる？」
```

**記録すべき情報**:
- 現在の計数方法
- 実施頻度（毎日？週1？）
- 実施者（店長？アルバイト？）
- 改善できない理由

### 2-5. 意思決定プロセス（3分） - 潜在的ニーズの発掘

```
「では、シフト計画や仕入れ量を決める時は、どのような情報を基に決めていますか？」

深掘り例：
1. シフト計画
   - 「何を基準に『この日は人多く配置する』って決めます？」
   - 「営業開始時点で『今日は混みそう』ってどうやってわかります？」

2. 仕入れ量
   - 「仕込み量・仕入れ量は？」
   - 「朝決める？前日決める？」
   - 「廃棄が出ることはありますか？」
   - 「品切れのモヤモヤはありますか？」

3. 営業施策（イベント、セール等）
   - 「キャンペーンをやるときは、効果をどうやって見てますか？」
   - 「うまくいった・いかなかったの判断は？」

⭐ 目的：「データなしに決めている＝データがあれば改善できる」を探る
```

### 2-6. テクノロジー確認（2分） - 技術的フィージビリティ

```
【カメラ環境**：重要
「防犯カメラは設置していますか？」

Yes の場合：
- 「どのメーカー/機種ですか？」
- 「何台くらい？」
- 「入口/レジ周り/店内全体など、どこを撮ってますか？」
- 「映像は保存してますか？」（クラウド？NAS？DVR？）

No の場合：
- 「防犯対策は何かしてますか？」
- 「カメラ導入を検討したことは？」
- 「費用面の懸念は？」

⭐ 注目：既設カメラ + 既にNAS/クラウド保存 = 理想的な顧客
```

### 2-7. 価値認識 & 価格感度（5分） - 最重要セクション2 ⭐⭐⭐

**進め方**: 段階的に機能を提示しながら反応を見る（段階的開示）

```
【段階1】基本的なデータ
「では例えば『毎日の来客数の推移』『時間帯別のお客さんの数』がわかるとしたら、
どんなことに使いたいですか？」

（反応を聞く）

例えば：
- 「シフト計画がもっと正確に立てられる」
- 「仕入れの廃棄を減らせる」
→ 「その効果って月額で何円くらいの価値ですか？」

【段階2】より詳細なデータ
「もし『性別・年代別の来客数』『滞在時間』『リピート率』もわかるとしたら？」

→ 「営業施策がもっと的確に打てる」なら：
  「例えば『シニア女性向けキャンペーンをやろう』という判断ができるということ？」

【段階3】予測機能
「さらに『明日の来客数予測』『トレンド分析』『競合比較』があったら？」

→ 反応を見て、本当に欲しい機能が何かを把握

【価格質問テンプレート】
「こういった機能があったら、月額いくらなら使ってみたいですか？」

価格帯を提示する場合：
- 「月額 5,000円、10,000円、20,000円 の中だったら？」
- 「初期導入費がかかるとしたら？」

⭐ 目的：顧客が『使いたい』と思う機能と価格帯を把握
```

**記録すべき情報**:
- 提示した機能のどれに反応したか
- 言及した価格帯
- 躊躇した点（何が懸念か）

---

### 2-8. 競合・代替案（2分）

```
「似たようなサービス、試したことはありますか？」

Yes の場合：
- 「どのサービス？」
- 「何が良かった？悪かった？」
- 「今も使ってますか？」
- 「だからやめた理由は？」

No の場合：
- 「検討したことは？」
- 「なぜ導入しなかった？」
（費用？操作が複雑？必要性を感じなかった？）

⭐ 学び：競合比較 or 『何もやってない理由』から、真の障壁を探る
```

## 3. 成功基準と判定基準

### GO判定（広告テスト開始）
```
✅ 以下のいずれかを満たすインタビューが3件以上

□ 来客数・来客パターンの把握が明確なペインとして言及された
□ 現在の計数方法（目視・売上推測など）の限界を認識している
□ 提示した機能に対して「使ってみたい」という言及がある
□ カメラが既設で、データ活用の可能性を認識
□ 月額5,000〜15,000円程度の価格帯で購買意欲の兆候あり
```

### NO-GO判定（プロダクト改善 or セグメント変更）
```
❌ 以下の場合

□ 5件とも「来客数データは不要」と判定
□ 3件以上がカメラ非設置 + 導入意欲もない
□ 提示した機能に反応がない、または「高い」という声ばかり
□ 競合（POS連携、既存分析ツール等）で既に満足している企業が多い
```

## 5. 募集方法 - 5件のお客様をどう見つけるか

### 5-1. 優先順位付き採用戦略

#### Tier 1：最優先（紹介 + 動機付け強い）
```
対象：知人経由の飲食店・小売店

メリット：
- 信頼関係がある
- インタビューに応じやすい
- フィードバック後も連絡取りやすい

アクション：
1. 経営者知人に「飲食店・小売店の経営者を紹介してほしい」と相談
2. 紹介時に：「新しいビジネスのリサーチを手伝ってくれませんか？」
3. インタビュー後：簡単なお礼（ドリンク / ギフト）
```

#### Tier 2：次優先（商店街 + 直接訪問）
```
対象：商店街の店舗（複数が集中）

メリット：
- 複数店舗に短時間で訪問できる
- 商店街事務局から紹介をもらえる

アクション：
1. 近所の商店街事務局に連絡
   「新しいサービスのリサーチをしていて、店舗経営者のお話を聞きたい」
2. 事務局から紹介をもらう or 直接訪問時に「店長さんいますか？」
3. 店舗に訪問時：営業ではなく「リサーチへの協力」として依頼
```

#### Tier 3：スケール用（web企業 + DM）
```
対象：Google My Business でランキング上位の飲食店・小売店

メリット：
- リストアップが簡単
- スケーラブル

アクション：
1. Google Map で「渋谷 ラーメン」「新宿 カフェ」などで検索
2. 営業時間確認 → 電話 or Instagram DM で「インタビュー依頼」
3. 成功率は低いが（10-15%）、数を増やしやすい
```

## 7. よくある質問 & 対応

### Q: インタビューで「営業」に見えないようにするには？

**A**:
- 「検討しませんか？」という提案は最後にする
- 「どうやってますか？」という『聞く姿勢』を優先
- 機能紹介は「反応を見るための段階的開示」として
- お礼はする（信頼構築）が、売上圧力を見せない

---

### Q: 価格を聞く時、相手が萎縮しないか？

**A**:
- 段階的に提示する（いきなり「月額は？」ではなく）
- 「使うなら」という仮定形で聞く
- 「あくまで参考までに」と枕詞を付ける
- 「予算がなければ無理」という了承をしておく

---

### Q: 来客データが不要だった場合は？

**A**:
1. **セグメント変更**: 業種を変更（例：小売 → 飲食に絞る）
2. **機能ピボット**: 来客数 → セキュリティ分析に変更
3. **別売上モデル**: B2B2C（商店街事務局向け）に変更
4. **価格リバランス**: 機能を削減して価格を下げる

---

### Q: インタビュー中に「これは使いたい！」って言われたら？

**A**:
- 「ありがとうございます。ベータ版ができたら連絡してもいい？」と確認
- 連絡先を記録
- 後から「実装時間 / ベータ開始時期」を明確に伝える
- 約束は守る（信用が次の販売につながる）

## 9. 参考：インタビュー分析の見方

### 「GO判定」の例

```
✅ インタビュー1（ラーメン屋・オーナー）
- ペイン：「営業時間ごとの来客数がわからず、シフトが適当になってる」
- 現状：目視で数えたり、感覚で決めてる
- カメラ：防犯カメラ既設（パナソニック、4台）
- 反応：「時間帯別のデータがあれば、確実にシフト最適化できる」
- 価格：「月5,000円なら試してみたい」
- 判定：✅ GO要因

✅ インタビュー3（百貨店・売場長）
- ペイン：「性別・年代別の来客数がほしい。営業企画の効果測定ができない」
- 現状：売上でしか判断できない
- カメラ：Axis の高級カメラ既設（15台以上）
- 反応：「デモグラフィックデータはかなり価値がある」
- 価格：「月10,000円までなら」
- 判定：✅ GO要因

⚠️ インタビュー2（コンビニFC）
- ペイン：「本部からの指示で動いている」
- 現状：本部の指示に従うだけ
- 反応：「個店での施策を打つ権限がない」
- 判定：⚠️ 判定材料だが GO要因ではない

❌ インタビュー4（小型美容室）
- ペイン：「顧客リストと顧客管理が課題」（来客数ではない）
- 現状：Salesforce で管理
- 反応：「来客数データはあまり必要ない」
- 判定：❌ NO-GO要因
```

### 分析フレームワーク

```
3面分析で GO/NO-GO 判定：

1️⃣ ペイン × データ関連度
   （来客データで解決可能か）
   高い：○ / 低い：✗

2️⃣ テクノロジー準備度
   （カメラ環境 + クラウド対応）
   高い：○ / 低い：✗

3️⃣ 購買力
   （月額価格感度 + ベータ試用意向）
   高い：○ / 低い：✗

3つ中 3つ ✅ → 強いGO要因
3つ中 2つ ○ → 判定材料
3つ中 1つ以下 → NO-GO要因
```

## 10. 最後に：インタビュアーのマインドセット

```
✅ DO（こうあるべき）
□ 相手の話を「聞く」ことに専念する
□ 相手の問題を「引き出す」ことに集中する
□ 「本当のニーズ」を見つけることに徹する
□ 長い沈黙を恐れない（考えている時間）
□ 意外な答えを大事にする

❌ DON'T（避けるべき）
□ 自社の機能を説明しすぎる
□ 「いいでしょ？」という説得的なトーン
□ 相手の話を遮ったり、先読みしない
□ 「来客数が必要ですよね？」という誘導
□ 結論ありきで聞く

🎯 心構え
「相手はプロダクトの審判官ではなく、教科書である」
→ 相手から学び、プロダクトを改善する。それが GO/NO-GO 判定に勝る。
```
