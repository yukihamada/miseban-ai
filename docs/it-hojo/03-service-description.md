## 1. サービス概要

### ミセバンAIとは

**ミセバンAI**は、既存の防犯カメラやスマートフォンをAI店長に変えるクラウド型店舗分析ソリューションです。

高価な専用機器を新たに購入する必要はありません。今お使いの防犯カメラの映像をクラウドAIがリアルタイムで分析し、**来客数カウント・客層分析・店内動線ヒートマップ・防犯アラート・AI経営アドバイス**を自動で提供します。分析結果はLINEやSlackに通知され、Webダッシュボードからいつでも確認できます。

### キャッチコピー

> **既存カメラが、AI店長に変わる。**

### 3つの特徴

| 特徴 | 内容 |
|------|------|
| **初期費用ゼロ** | 既存カメラをそのまま利用。専用機器の購入不要、工事不要 |
| **月額0円から** | フリープランで基本機能を無料提供。14日間無料トライアル |
| **かんたん導入** | Docker、スマートフォン、Raspberry Piの3つの方法で最短5分導入 |

## 4. システム構成

### アーキテクチャ

```
              既存カメラ / スマートフォン
                       |
                 RTSP / MJPEG / HTTP
                       |
                       v
            +---------------------+
            |   ミセバンAI Agent   |   <-- エッジ側（PC/RPi/Phone）
            |  (フレームキャプチャ)  |       映像はここで処理
            +---------------------+
                       |
                    HTTPS（暗号化）
                    メタデータのみ送信
                       |
                       v
            +---------------------+
            |  クラウドAI (Fly.io) |   <-- 日本リージョン(東京)
            |  - Analysis API     |
            |  - Dashboard        |
            |  - Data Store       |
            +---------------------+
                    /        \
                   v          v
          +----------+   +----------+
          |Dashboard |   |LINE/Slack|
          |  (Web)   |   |   通知    |
          +----------+   +----------+
```

### エッジ処理の特長

- **プライバシー保護**: 生映像はクラウドに保存されない。分析結果（メタデータ）のみ送信
- **低帯域**: 画像1フレーム（JPEG）のみを定期送信。常時ストリーミング不要
- **オフライン耐性**: SQLiteバッファにより、ネットワーク断時もフレームをローカル保存し、復旧後に自動再送

### 3つの導入方法

| 方法 | 対象 | セットアップ時間 | 特徴 |
|------|------|-----------------|------|
| **PC（Docker）** | Windows / Mac / Linux | 約5分 | 最も簡単。Docker対応PCで即稼働 |
| **スマートフォン** | iPhone / Android | 約3分 | アプリをインストールしてQRコード読み取り |
| **Raspberry Pi** | Raspberry Pi Zero 2 W | 約10分 | 超小型・低コスト・常時稼働に最適 |

## 8. セキュリティ・プライバシー

### データ保護方針

| 項目 | 対策 |
|------|------|
| 映像データの保管 | **クラウドに生映像は保存しない**。エッジ側で処理し、メタデータのみ送信 |
| 通信の暗号化 | 全通信はHTTPS/TLS暗号化 |
| 認証方式 | JWT（JSON Web Token）ベースのAPI認証 |
| アクセス制御 | 店舗ごとのデータ分離。他店舗データへのアクセス不可 |
| 個人情報の取扱い | 個人を特定する情報は保存しない（統計データのみ） |

### セキュリティ監査体制

| ツール | 用途 |
|--------|------|
| cargo-deny | 依存ライブラリのライセンス・脆弱性チェック |
| gitleaks | ソースコード内のシークレット漏洩防止 |
| trivy | コンテナイメージの脆弱性スキャン |
| cosign | コンテナイメージの署名・検証 |
| SBOM | ソフトウェア部品表の自動生成・管理 |

## 付録A: 用語集

| 用語 | 説明 |
|------|------|
| **エージェント (Agent)** | カメラからフレームを取得し、クラウドに送信するソフトウェア。PC、スマホ、RPi上で動作 |
| **フレーム** | カメラから取得した1枚の静止画像（JPEG形式） |
| **ヒートマップ** | 店舗内の人の動きを色の濃淡で可視化した図。赤=滞留が多い、青=少ない |
| **RTSP** | Real Time Streaming Protocol。IPカメラの映像をネットワーク経由で取得するプロトコル |
| **ONVIF** | IPカメラの共通規格。異なるメーカーのカメラでも統一的に操作可能 |
| **YOLOv8** | 最新の物体検出AIモデル。画像内の人物をリアルタイムで高精度に検出 |
| **エッジ処理** | データをクラウドに送らず、ローカル（現場のデバイス上）で処理すること |
| **SaaS** | Software as a Service。クラウド経由で提供されるソフトウェアサービス |

## 付録B: よくある質問（FAQ）

**Q1. 既存のカメラがそのまま使えますか？**
A. はい。RTSP/ONVIF対応のIPカメラであれば、20ブランド以上・数百モデルに対応しています。お使いのカメラが対応しているかは https://misebanai.com/cameras.html で確認できます。

**Q2. インターネット環境は必要ですか？**
A. はい。エージェントからクラウドへの通信にインターネット接続が必要です。ただし、一時的なネットワーク断の場合はローカルにバッファリングし、復旧後に自動再送します。

**Q3. 映像データはクラウドに保存されますか？**
A. いいえ。生映像はクラウドに保存されません。エージェント側でフレームを処理し、分析結果（来客数などのメタデータ）のみがクラウドに送信されます。

**Q4. 個人情報の取扱いは大丈夫ですか？**
A. 個人を特定する情報は一切保存しません。来客数・属性推定（年齢層・性別の統計値）のみを扱います。顔認識（個人特定）機能は搭載していません。

**Q5. 導入にどれくらい時間がかかりますか？**
A. 最短5分です。Dockerを使えばコマンド1つで起動できます。セットアップウィザード付きで、ITに詳しくない方でも画面の案内に沿って設定できます。

**Q6. 解約はいつでもできますか？**
A. はい。月額プランのため、いつでも解約可能です。解約違約金はありません。

**Q7. IT導入補助金を使って導入できますか？**
A. はい。デジタル化・AI導入補助金2026の通常枠で申請可能です。補助率1/2〜2/3で、年間費用の半分以上が補助されます。
