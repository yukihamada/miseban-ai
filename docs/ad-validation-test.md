# 広告バリデーションテスト計画

## 目的

**¥30,000で「ミセバンAIに需要があるか」を7日間で検証する。**

実際にプロダクトを作り込む前に、Google Ads で検索広告を出し、ランディングページへのメール登録数を計測することで、市場の需要を定量的に判断する。

- ランディングページURL: https://miseban-ai.fly.dev（仮）/ https://miseban.ai（ドメイン設定後）

## Google Ads キャンペーン設定

### アカウント設定手順

1. [ads.google.com](https://ads.google.com) にアクセスしてログイン
2. 「+新しいキャンペーン」をクリック
3. キャンペーン目標: **リード獲得**
4. キャンペーンタイプ: **検索**
5. 日予算: **¥4,300/日**（7日間で約¥30,000）
6. 地域ターゲティング: **日本全国**（初期設定。反応を見て都市部に絞る可能性あり）
7. 言語: **日本語**
8. 入札戦略: **コンバージョン数の最大化**（最初の2日はデータ収集のため「クリック数の最大化」でも可）
9. ネットワーク: Google検索のみ（ディスプレイネットワークはオフ）

### 広告文（各グループ3パターン）

各広告グループに以下の3パターンを設定し、A/Bテストを実施する。Google Ads のレスポンシブ検索広告の仕組みで自動最適化される。

#### パターンA（価格訴求）

- **見出し1**: 店舗分析AI 月額2,980円〜
- **見出し2**: 既存カメラそのまま｜工事不要
- **見出し3**: IT導入補助金で実質半額
- **説明文**: 今あるカメラがAI店長に。来客カウント・属性分析・防犯アラートをLINEでお届け。初期費用0円・5分で設置完了。まずは無料プランでお試し。

#### パターンB（課題訴求）

- **見出し1**: 来客数、正確に把握できてますか？
- **見出し2**: AIが自動カウント｜月額2,980円
- **見出し3**: 無料で始められます
- **説明文**: 「今日何人来たか」がLINEで届く。既存の防犯カメラにAIをプラスするだけ。来客数・ピーク時間・客層がひと目でわかる。大手サービスの1/5の価格。

#### パターンC（競合比較訴求）

- **見出し1**: 店舗分析、高すぎませんか？
- **見出し2**: 月額16,000円→2,980円に
- **見出し3**: 既存カメラ対応｜工事不要
- **説明文**: 大手店舗分析サービスの1/5の価格。今あるカメラをそのまま使えるから初期費用0円。来客カウント・ヒートマップ・防犯を月額2,980円で。IT補助金対象。

### コンバージョン設定

ランディングページのメール登録フォーム送信をコンバージョンとして計測する。

1. Google Ads → **ツールと設定** → **コンバージョン** → **新しいコンバージョンアクション**
2. 種類: **ウェブサイト**
3. イベント名: `generate_lead`
4. コンバージョン値: **¥3,000**（見込み顧客1件あたりの想定価値）
5. カウント方法: **初回のみ**（同一ユーザーの重複送信を除外）
6. アトリビューション: クリックスルーコンバージョンの計測期間は30日

**実装方法（LP側）**:

フォーム送信完了時に以下のイベントを発火させる。

```html
<!-- Google Ads コンバージョンタグ（<head>内に設置） -->
<script async src="https://www.googletagmanager.com/gtag/js?id=AW-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'AW-XXXXXXXXXX');
</script>

<!-- フォーム送信完了時に呼び出す -->
<script>
  function trackConversion() {
    gtag('event', 'conversion', {
      'send_to': 'AW-XXXXXXXXXX/XXXXXXXXXXXXXXXXXXX',
      'value': 3000,
      'currency': 'JPY'
    });
  }
</script>
```

> `AW-XXXXXXXXXX` は Google Ads 管理画面から取得した実際のコンバージョンIDに置き換える。

## 並行してやること

### Day 1-3: 定性データ収集

登録があったら**24時間以内**に電話またはメールでコンタクトする。初期のリードは宝の山であり、定量データだけでは見えないインサイトを得る最大のチャンス。

#### ヒアリング項目

1. **現状把握**: 今どうやって来客数を把握していますか？（手動カウント / 勘 / 既存ツール / 把握していない）
2. **予算感**: 月にいくらまでなら店舗分析ツールに払えますか？
3. **優先機能**: 一番欲しい機能は何ですか？（来客カウント / 属性分析 / ヒートマップ / 防犯 / LINE通知）
4. **カメラ環境**: カメラは何台ありますか？ メーカー・型番は？ ネットワーク接続されていますか？
5. **意思決定**: 導入の意思決定は誰がしますか？（本人 / 店長 / 本部）
6. **業種・規模**: 業種と店舗数を教えてください

#### ヒアリング記録先

Google Sheetsなどに以下のカラムで記録する。

```
日付 | 名前 | メール | 電話番号 | 業種 | 店舗数 | カメラ台数 | 現状の把握方法 | 月額予算感 | 優先機能 | 備考
```

### Day 4-7: データ分析

定量データと定性データを突き合わせて、以下の問いに答える。

- **どのキーワードグループが一番CVR（コンバージョン率）が高いか？**
  → 需要の強さと緊急度がわかる
- **どの広告文が一番CTR（クリック率）が高いか？**
  → どの訴求が刺さるかがわかる
- **どの業種（フォームの回答から）が一番多いか？**
  → 初期ターゲットセグメントが明確になる
- **電話ヒアリングの共通パターンは何か？**
  → プロダクトの優先機能が明確になる

## テスト後の判断フレームワーク

### GO判定（50件以上）

需要は十分にある。プロダクト開発に全力投球する。

- 即座にMVP開発開始（4週間スプリント）
- 登録者全員にβ版先行案内メールを送信
- ヒアリングで反応の良かった業種にフォーカス
- 代理店候補（設置代行パートナー）にもアプローチ開始
- 追加広告予算を検討（¥100,000/月〜）

### PIVOT判定（10〜49件）

需要の兆候はあるが、訴求やターゲットが合っていない可能性がある。

- どのキーワード / 広告文が反応良かったか詳細分析
- ヒアリング結果から訴求ポイントを修正
- LP（ランディングページ）のコピー・デザインを改修
- 追加¥30,000でリテスト（2週目）
- 2回目も不振なら STOP 判定へ

### STOP判定（10件未満）

「店舗AI分析」自体への需要が薄い、または現時点ではオンライン広告でリーチできる層が少ない可能性が高い。

- ヒアリングで「なぜ登録しなかったか」をLPの離脱分析から把握
- Google Search Console で検索ボリュームを再調査
- 別のアプローチを検討:
  - 別アイデア: AIレジ、AI電話応答、AI在庫管理 等
  - 別チャネル: 飲食店コミュニティ直接営業、展示会出展、業界メディアPR
  - 別ターゲット: 店舗オーナーではなくカメラ設置業者（代理店モデル先行）
