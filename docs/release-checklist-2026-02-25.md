## 緊急対応プレイブック

### サービスダウン時

1. Fly.ioダッシュボードでステータス確認
2. `fly status -a miseban-web` / `fly logs -a miseban-web` でログ確認
3. 必要に応じてv26へロールバック: `fly deploy -a miseban-web --image <v26-image>`
4. ステータスページ / SNSで障害告知
5. 復旧後、原因分析と再発防止策を記録

### 大量アクセスによる性能劣化時

1. Fly.ioのオートスケーリング設定を確認・調整
2. `fly scale count` でインスタンス数を増加
3. CDN（静的アセット）のキャッシュ設定を確認
4. 必要に応じてレートリミットを調整

### セキュリティインシデント時

1. 影響範囲の特定
2. 該当エンドポイント / 機能の一時停止
3. ログの保全
4. 修正パッチの適用
5. 影響を受けたユーザーへの通知
