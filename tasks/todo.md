## Phase 1: MVP構築 (Day 1-30)

### Week 1-2: 基盤構築 (Day 1-14)

#### インフラ・開発環境
- [ ] GitHub リポジトリ整備（ブランチ保護、CI/CD テンプレ）
- [ ] モノレポ構成確定（`crates/agent`, `crates/inference`, `crates/api`, `web/`）
- [ ] Docker Compose で開発環境一発起動（API + DB + 模擬カメラ）
- [ ] Supabase プロジェクト作成（Auth, DB, Storage, Edge Functions）
- [ ] DB スキーマ設計・マイグレーション（stores, cameras, events, analytics, users）
- [ ] Terraform / Pulumi で AWS インフラ定義（ECS, S3, SQS, CloudWatch）

#### ランディングページ・ウェイトリスト
- [ ] Next.js ランディングページ作成（LP デザイン・コピーライティング）
- [ ] ウェイトリスト登録フォーム（Supabase + メール通知）
- [ ] OGP / SEO 最適化
- [ ] Google Analytics / Plausible 導入
- [ ] Fly.io or Vercel にデプロイ

> **Milestone Day 7: ランディングページ公開、ウェイトリスト受付開始**

#### エッジエージェント（カメラ接続）
- [ ] Rust エージェントの骨格実装（`crates/agent`）
- [ ] RTSP クライアント実装（GStreamer or ffmpeg バインディング）
- [ ] テスト用 RTSP サーバー構築（Docker、サンプル動画ループ再生）
- [ ] フレーム取得 → JPEG エンコード → S3 アップロードパイプライン
- [ ] エージェント ↔ クラウド間の WebSocket / MQTT 通信プロトコル設計
- [ ] エージェントの自動起動・ヘルスチェック・再接続ロジック
- [ ] 実カメラ（HIKVISION / Dahua）での接続テスト

> **Milestone Day 14: エージェントが実 RTSP カメラに接続し、フレームをクラウドに送信**

### Week 3-4: 推論・ダッシュボード (Day 15-30)

#### YOLO 推論パイプライン
- [ ] YOLOv8/v9 モデル選定（人物検出 + 姿勢推定）
- [ ] ONNX Runtime or TensorRT での推論サーバー実装（`crates/inference`）
- [ ] AWS ECS (GPU) or Lambda + SageMaker エンドポイント構成
- [ ] SQS キューイング: フレーム到着 → 推論リクエスト → 結果保存
- [ ] 推論結果スキーマ: `{timestamp, camera_id, person_count, zones, heatmap_data}`
- [ ] バッチ処理: 1fps サンプリングで推論コスト最適化
- [ ] 推論精度のベンチマーク（mAP 測定、閾値チューニング）

> **Milestone Day 21: YOLO 推論がクラウドで稼働、人数カウント結果を DB に保存**

#### API サーバー
- [ ] Axum ベースの REST API 実装（`crates/api`）
- [ ] 認証: Supabase Auth JWT 検証ミドルウェア
- [ ] エンドポイント設計・実装:
  - [ ] `GET /api/v1/stores` — 店舗一覧
  - [ ] `GET /api/v1/stores/:id/analytics` — 来客数・滞在時間・ヒートマップ
  - [ ] `GET /api/v1/stores/:id/cameras` — カメラ一覧・ステータス
  - [ ] `GET /api/v1/stores/:id/events` — アラート・イベント一覧
  - [ ] `POST /api/v1/stores/:id/cameras` — カメラ登録
  - [ ] `GET /api/v1/dashboard/summary` — ダッシュボード集計
- [ ] レートリミット・CORS 設定
- [ ] API ドキュメント（OpenAPI / Swagger）

#### Web ダッシュボード
- [ ] Next.js + Tailwind CSS + shadcn/ui でダッシュボード構築
- [ ] 認証画面（ログイン / サインアップ / パスワードリセット）
- [ ] ダッシュボードホーム: 本日の来客数、前日比、時間帯グラフ
- [ ] リアルタイム来客カウンター（WebSocket or SSE）
- [ ] ヒートマップ表示（店内の動線可視化）
- [ ] カメラ管理画面（RTSP URL 登録、ステータス確認、サムネイル表示）
- [ ] 日次 / 週次 / 月次レポートビュー
- [ ] モバイルレスポンシブ対応

> **Milestone Day 30: MVP 完成。ダッシュボードでリアルタイム来客数・ヒートマップ確認可能。Beta 募集開始**

## リスク・課題

| リスク | 影響 | 対策 |
|--------|------|------|
| カメラ互換性問題 | 高 | ONVIF 標準準拠、主要メーカー事前テスト |
| 推論コスト超過 | 中 | フレームレート最適化、エッジ推論の段階的導入 |
| プライバシー懸念 | 高 | 顔画像非保存、匿名化処理、法務レビュー |
| Beta 店舗確保 | 中 | 初期無料提供、既存ネットワーク活用 |
| ネットワーク環境 | 中 | エージェント側バッファリング、低帯域モード |
